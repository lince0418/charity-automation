# LINE 圖片推播流程 SOP

## 目前測試結果

| 項目 | 狀態 |
|---|---|
| 官方帳號文字推播 | 已成功 |
| 官方帳號圖片推播 | 已成功 |
| 測試群組 | 自己喔 |
| 圖片測試方式 | 使用公開 HTTPS PNG 圖片 |

## 重要限制

LINE Messaging API 發送圖片時，圖片必須是公開 HTTPS 網址。

不能直接使用：

```text
/Users/singyuan/Desktop/...
```

必須使用：

```text
https://...
```

## 建議正式流程

| 步驟 | 說明 |
|---|---|
| 1 | 組員在 Canva 完成圖片文宣 |
| 2 | 你確認日期、主題、講師、人物照片排版 |
| 3 | 從 Canva 下載 `.png` 或 `.jpg` |
| 4 | 放到對應月份資料夾 |
| 5 | 將圖片上傳到可公開讀取的位置 |
| 6 | 取得 HTTPS 圖片網址 |
| 7 | 系統使用 LINE API 發送文字 + 圖片 |
| 8 | 工作日誌記錄發送結果 |

## 圖片存放建議

本機資料夾：

```text
協會演講/2026年/月份/宣傳素材/
```

公開連結紀錄：

```text
協會演講/2026年/月份/LINE圖片連結.md
```

## 圖片網址來源選項

| 方式 | 建議度 | 說明 |
|---|---:|---|
| Google Drive 公開圖片連結 | 中 | 需要轉換成可直接讀圖的網址，權限設定容易出錯 |
| Canva 直接公開連結 | 中 | 適合人工分享，不一定適合 LINE API 圖片訊息 |
| 專用圖片空間或網站 | 高 | 最穩，能提供直接圖片網址 |
| 暫時測試圖片網址 | 高 | 適合測試，不適合正式活動 |
