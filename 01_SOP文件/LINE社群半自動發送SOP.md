# LINE 社群半自動發送 SOP

## 結論

目前不建議用私人 LINE 帳號做全自動發送。

原因是你目前是在 LINE 社群中，以私人帳號加入，並透過修改頭銜或顯示名稱來代表協會發文。這種情境沒有穩定、正式的官方 API 可以直接代替私人帳號在 LINE 社群中發送文字與圖片。

最適合目前狀況的做法是：

> 系統自動產生文字、找出對應圖片、整理成待發送包，並在指定時間提醒你手動貼到 LINE 社群。

## 目前需求

| 項目 | 說明 |
|---|---|
| 發送平台 | LINE 社群 |
| 發送身份 | 私人 LINE 帳號，但在社群內改成協會相關頭銜或名稱 |
| 發送內容 | 文字文案 + 該主題圖片文宣 |
| 發送頻率 | 固定週期 |
| 是否可全自動發送 | 不建議 |
| 建議作法 | 半自動：自動備稿、自動找圖、自動提醒、手動發送 |

## 發送週期

| 文案類型 | 發送時間 | 內容 |
|---|---|---|
| 月預告 | 當月月底 | 下個月兩場線上公益講座預告 |
| 教練講座預告 | 發送日當週星期一 | 教練講座文字 + 教練講座圖片文宣 |
| 成功人士預告 | 發送日當週星期一 | 成功人士文字 + 成功人士圖片文宣 |
| 上線日通知 | 活動當天 | 上線通知文字 + 該主題圖片文宣 |

## 建議資料夾整理方式

每個月份建議固定保留以下資料：

| 檔案或資料夾 | 用途 |
|---|---|
| 自動產生文案.md | 系統產生的月預告、講座預告、成功人士預告、上線日通知 |
| 宣傳素材 | 放該月圖片文宣 |
| 待發送包 | 放當週或當日要手動發送到 LINE 社群的文字與圖片 |

建議範例：

```text
協會演講/2026年/七月份/
├── 自動產生文案.md
├── 宣傳素材/
│   ├── 2026-07-09_教練講座_楊家欣.jpg
│   └── 2026-07-23_成功人士_邱凡鳴.jpg
└── 待發送包/
    ├── 2026-07-06_星期一_教練講座預告.md
    ├── 2026-07-06_星期一_教練講座預告_圖片.jpg
    ├── 2026-07-09_當天_上線日通知.md
    └── 2026-07-09_當天_上線日通知_圖片.jpg
```

## 半自動流程

| 步驟 | 系統處理 | 你需要做的事 |
|---|---|---|
| 1 | 每天檢查 Google 試算表 | 確認日期、主題、講師資料有填好 |
| 2 | 自動產生文案 | 檢查文字是否符合當月需求 |
| 3 | 依活動日期與類別整理待發送包 | 檢查圖片是否正確 |
| 4 | 到發送日提醒 | 開啟待發送包 |
| 5 | 複製文字並上傳圖片到 LINE 社群 | 手動發送 |
| 6 | 工作日誌記錄「已備稿」或「待手動發送」 | 發送後可補註已完成 |

## 為什麼不建議全自動

| 風險 | 說明 |
|---|---|
| 私人帳號沒有正式發送 API | LINE 官方自動發送主要支援官方帳號，不支援私人帳號代發 |
| LINE 社群不等同一般群組 | 官方帳號 Messaging API 對一般群組較清楚，LINE 社群不一定能同樣操作 |
| 模擬手機或電腦操作不穩定 | 只要 LINE 畫面、登入狀態、驗證或按鈕位置改變，就可能失敗 |
| 帳號安全風險 | 私人帳號若被判定異常操作，可能影響使用 |
| 圖片上傳更容易失敗 | 圖片檔案、視窗焦點、網路狀態都會影響模擬發送 |

## 建議的最佳落地方案

### 第一階段：半自動備稿

先把工作做到這個程度：

- 自動產生文案
- 自動找出對應圖片
- 自動整理待發送包
- 自動寫入工作日誌
- 到指定時間提醒你手動發送

這階段最穩，也最符合你目前使用私人帳號在 LINE 社群發文的現況。

### 第二階段：協會官方帳號

若未來協會願意改用 LINE 官方帳號，並且社群或群組可以接受官方帳號進入，就可以評估：

- 官方帳號自動推播
- 訊息發送紀錄
- 發送失敗回報
- 圖文訊息模板
- 多平台同步發送

但這會牽涉協會帳號、權限、LINE 官方帳號方案與社群是否支援。
