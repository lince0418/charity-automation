# 圖片增量自動化巡檢模板

## 執行順序

| 步驟 | 動作 | 結果 |
|---|---|---|
| 1 | 執行 `previewLectureImageFolder` | 先看會處理哪些檔案 |
| 2 | 檢查 `圖片處理日誌` | 確認是否有 `待確認` |
| 3 | 執行 `scanLectureImageFolder` | 正式增量處理 |
| 4 | 檢查 `LINE 圖片網址清單` | 確認圖片網址已產生 |
| 5 | 執行月份模擬 | 確認文案任務是否都有圖片 |

## 驗收標準

| 項目 | 通過條件 |
|---|---|
| 已完成圖片 | 不重新改名、不重複處理 |
| 新圖片 | 能判斷就自動改名並產生連結 |
| 不明圖片 | 標記 `待確認` |
| LINE 圖片網址 | 有 `https://drive.google.com/uc?export=view&id=` |
| 月份模擬 | 所有待發送任務顯示 `有圖片網址` |

## 待確認處理

若 `圖片處理日誌` 出現 `待確認`：

1. 看原檔名。
2. 判斷它屬於哪一場活動。
3. 手動改成標準命名。
4. 再執行 `scanLectureImageFolder`。
